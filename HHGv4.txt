Changelog (v3 → v4)

Fix 1 — γ Double-Counting Bug (Critical)

Problem (v3): _build_lindblad_ops already multiplied each collapse operator by sqrt(γ). Then run() did:

c_ops = [qt.sqrt(self.config.gamma) * L for L in self.ops["L_list"]] 

This meant the effective Lindblad rate was γ² instead of γ — silently wrong for any γ ≠ 1.

Fix (v4): run() now passes self.ops["L_list"] directly to mesolve/mcsolve with no extra factor. _build_lindblad_ops remains the single authoritative location for sqrt(γ).

Fix 2 — Auto-mcsolve for Large N

When use_mcsolve=None (default) and N > 8, the solver now automatically switches to mcsolve with a clear printed message. The user can suppress this by passing use_mcsolve=False or use_mcsolve=True explicitly.

The N > 8 warning message now also mentions the use_quimb=True option.

Fix 3 — Time-Dependent Dissipation

New SimConfig.gamma_t: Optional[Callable[[float], float]] field. When set, run() steps through the time grid manually, calling _build_lindblad_ops(gamma_override=gamma_t(t)) at each step and running single-interval mesolve segments. This supports:

Laser-induced heating (monotonically rising γ(t))

Pump-probe bath switching

Non-equilibrium bath protocols

Fix 4 — Momentum-Space Observables

Two new methods on HubbardHHGSimulator:

MethodReturns_bz_k_points()BZ k-point grid (lattice-type aware)_compute_momentum_density(rho)n(k) = Σ_j exp(-ik·r_j) ⟨n_j⟩_compute_momentum_current(τ, rho)J(k,t) — Fourier-transformed bond current 

Activated in run() via track_momentum=True (requires periodic=True). Results stored in result["n_k_t"] and result["J_k_t"]. A plot_momentum_density() convenience function is also provided.

Fix 5 — Floquet / Intensity Scan

New top-level function floquet_intensity_scan(base_config, E0_values, harmonic_orders) runs the full simulator for each E0 value and interpolates the HHG spectrum at each harmonic. Returns a dict with yields[i, j] = HHG yield at E0_values[i] and harmonic_orders[j]·ω₀. Companion plot_intensity_scan() function produces a log-log yield vs. intensity plot.

Fix 6 — Richer Pulse Control

Three pulse envelopes now available via SimConfig.pulse_envelope:

ValueEnvelope f(t)Notes"sin2"sin²(πt/T)Default (v3 behaviour)"gaussian"exp(-(t-T/2)²/(2σ²))Width set by sigma_cycles"few_cycle"cos²(πt/T)Hard-cutoff few-cycle 

Carrier-envelope phase phi_cep [rad] is applied to the sin(ωt + φ_CEP) carrier in _vector_potential(), enabling CEP-sensitive strong-field observables.

Fix 7 — save_lattice_plot()

New method on HubbardHHGSimulator:

sim.save_lattice_plot("my_lattice.png", dpi=200) 

Creates a Matplotlib figure, calls plot_lattice(), saves to the specified path, and closes the figure cleanly.

Fix 8 — quimb Tensor-Network Hook

New SimConfig.use_quimb: bool = False. When True, run() routes through _run_quimb(), which:

Checks for the quimb package and raises an informative ImportError if absent.

Provides a minimal dense-mode Euler propagator as a proof-of-concept.

Documents how to replace the Euler step with quimb's TEBD engine for actual MPS simulation.

Known Limitations / Future Work

Scalability: For N > 8–10, the QuTiP full density-matrix approach is intractable. The use_quimb=True hook currently uses dense Euler propagation; a true MPO-TDVP backend requires a more complete quimb integration.

Half-filling + electron_phonon bath: The electron_phonon model does not conserve N. Using half_filling=True enforces half-filling in the initial state but density will drift during evolution (physically meaningful for polaron physics).

Momentum observables: For non-square or non-periodic lattices the k-grid construction is approximate. Full BZ sampling requires a proper supercell reciprocal-lattice construction.

Time-dependent γ stepping: The manual step-by-step mesolve loop is correct but slow. A Lindblad super-operator with explicit time dependence passed to mesolve as a [H, args] list would be faster.
